UsmAudioCli — CRI USM demux console

Что это
-------
Лёгкий консольный “миниклон” VGMToolbox для демультиплексирования CRI USM.
Извлекает аудио-потоки в .hca (с обрезкой служебных секций как в VGMToolbox)
и по желанию видеопотоки.

Требования
----------
- .NET 8 SDK


Сборка
------
dotnet build -c Release
Исполняемый файл: bin\Release\net8.0\UsmAudioCli.exe

Использование
-------------
UsmAudioCli <input.usm> [--out DIR] [--audio] [--video] [--split] [--addheader]

Параметры:
  --out DIR     каталог, куда писать результаты (по умолчанию — рядом с .usm)
  --split       разносить аудиопотоки по stream_id       (audio0.bin, audio1.bin…)
  --addheader   добавлять заголовки поверх аудио         (обычно не требуется)

Примеры
-------
1) Только аудио, вывод рядом с файлом:
   UsmAudioCli.exe <input.usm>.usm --audio --split

Выходные файлы
--------------
- Аудио:  <имя USM>.audio{N}.hca
  Внутри может быть HCA / ADX / AIX / PCM. Расширение оставляем .hca, как в VGMToolbox.

Трюблшутинг
-----------
[warn] аудиопотоки не найдены
  — файл не USM, нестандартная разметка или извлекаете только видео без --video.

“Checksum does not match” в hca2wav
  — почти всегда неверный ключ/порядок. Попробуйте -k или смену порядка -b/-a.

Лицензия/заметки
----------------
Код повторяет логику VGMToolbox для CRI USM (демульс и резка служебных секций).
Инструмент предназначен для анализа/демульса медиа-файлов.
